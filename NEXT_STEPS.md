# Что делать дальше (практичный план)

Ниже — приоритетный план, чтобы довести проект до продакшена без хаоса.

## 1) Сначала запустить проект локально

```bash
cp .env.example .env.local
npm install
npm run dev
```

**Ожидаемый результат:** сайт открывается на `http://localhost:3000`, навигация между главной, услугами, блогом и контактами работает.

---

## 2) Обновить бизнес-данные (обязательно)

### Что заменить сразу
- Название/позиционирование на главной (`app/page.tsx`)
- Контактные данные (`app/contact/page.tsx`)
- SEO-description/OG (`app/layout.tsx`)
- URL домена в `.env.local`

```env
NEXT_PUBLIC_SITE_URL=https://your-real-domain.com
NEXT_PUBLIC_PLAUSIBLE_DOMAIN=
```

**Ожидаемый результат:** метаданные и контакты соответствуют вашему реальному бизнесу.

---

## 3) Привести контент к реальным офферам

Обновить MDX-файлы:
- `content/services/*.mdx`
- `content/blog/*.mdx`

Минимум для каждой услуги:
- результат для клиента,
- сроки,
- стоимость/диапазон,
- CTA.

**Ожидаемый результат:** сайт выглядит как коммерческий, а не как шаблон.

---

## 4) Подключить получение заявок (сейчас лог в консоль)

Сейчас `/api/contact` пишет заявку в `console.info`.
Для продакшена добавьте отправку в email/CRM (например Resend, Brevo, Pipedrive).

Минимальный путь:
1. Выбрать сервис отправки email.
2. Добавить серверный ключ в `.env` (без `NEXT_PUBLIC_`).
3. В `app/api/contact/route.ts` заменить `console.info` на реальную отправку.

**Ожидаемый результат:** после отправки формы лид приходит в почту/CRM.

---

## 5) Усилить антиспам перед рекламным трафиком

Сейчас есть:
- honeypot,
- in-memory rate limit.

Для продакшена лучше:
- Redis rate limit (Upstash),
- при необходимости Cloudflare Turnstile.

**Ожидаемый результат:** форма выдерживает ботов и частые запросы.

---

## 6) Проверить SEO и индексацию

Проверить вручную:
- `/sitemap.xml`
- `/robots.txt`
- Open Graph превью (через валидатор)

Добавить сайт в:
- Google Search Console,
- Bing Webmaster Tools.

**Ожидаемый результат:** страницы индексируются и корректно шарятся в соцсетях/мессенджерах.

---

## 7) Подключить аналитику

Если используете Plausible:
```env
NEXT_PUBLIC_PLAUSIBLE_DOMAIN=your-real-domain.com
```

**Ожидаемый результат:** просмотры страниц и события видны в аналитике.

---

## 8) Деплой на Vercel

1. Push в GitHub.
2. Import в Vercel.
3. Добавить env variables.
4. Deploy.

После деплоя:
- проверить форму,
- проверить 404/500,
- прогнать Lighthouse на главной и странице услуги.

**Ожидаемый результат:** production URL стабильно работает, Lighthouse ~90+ при оптимизированных ассетах.
